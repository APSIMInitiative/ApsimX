## GenericFruitTree scope

`GenericFruitTree` is a generic perennial fruit-tree model intended to represent the main structural and physiological behaviours needed for orchard simulations without being parameterised to a single fruit crop or cultivar. The current implementation focuses on generic tree canopy geometry, phenology, fruit growth, crop load, pruning/thinning responses, reserve storage and mobilisation, post-maturity fruit fate, water response, and fruit-quality diagnostics.

Crop-specific parameterisation and empirical validation are expected to be future work. The included example and sensibility tests are intended to demonstrate model operation, internal consistency, and biologically sensible responses under controlled generic orchard scenarios.

## Example simulation

The canonical example is:

```bash
dotnet ./bin/Debug/net8.0/Models.dll Examples/Agroforestry/GenericFruitTreeGenericOrchardExample.apsimx --verbose
```

The example is configured as a generic orchard scaffold rather than a crop-specific validation case.

## PerennialMicroClimate

`PerennialMicroClimate` and `PerennialMicroClimateZone` are adapted from the standard APSIM `MicroClimate` components for use with perennial tree canopies. It is my understanding that the standard `MicroClimate` implementation distributes canopy LAI uniformly through canopy depth. `GenericFruitTree` instead provides a precomputed canopy `LightProfile`, allowing radiation interception, LAI layering, and water-demand calculations to use the tree-specific vertical canopy profile while retaining the existing microclimate energy-balance calculations.

These classes are added separately to avoid changing the behaviour of existing APSIM crop models in this PR. If the approach is accepted, a later refactor could generalise the standard `MicroClimate` pathway to accept an optional externally supplied canopy light profile.

## Sensibility Test Coverage Overview

The sensibility suite is defined in `Tests/Validation/Agroforestry/GenericFruitTree/sensibility/GenericFruitTreeSensibility.apsimx`. It keeps one canonical orchard simulation and uses APSIM replacement/factorial machinery to run explicit named scenario levels, grouped by scenario intent rather than by cross-product combinations.

Run the GenericFruitTree sensibility suite:

```bash
DOTNET_CLI_HOME=/tmp/dotnet-cli-home DOTNET_SKIP_FIRST_TIME_EXPERIENCE=1 /usr/local/share/dotnet/dotnet ./bin/Debug/net8.0/Models.dll Tests/Validation/Agroforestry/GenericFruitTree/sensibility/GenericFruitTreeSensibility.apsimx --verbose
```

| Scenario group              | Simulations                                                                    | What it tests                                                                                                                                              |
| --------------------------- | ------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Baseline`                  | `Baseline`                                                                     | Verifies that the canonical orchard simulation runs cleanly and produces positive leaf growth, fruit biomass, and fruit-quality diagnostics.                |
